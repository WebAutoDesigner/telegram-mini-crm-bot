# Финальная спецификация Telegram Mini CRM Bot

## 1. Назначение

Бот — это личная mini CRM в Telegram для одного бизнеса, куда попадают все лиды из сайта и ручного ввода. Его главная задача — не терять заявки, быстро фиксировать результат контакта, вести записи и давать понятный план работы на сегодня и завтра.

## 2. Источники лидов

- Лиды приходят с сайта.
- Лиды можно добавлять вручную в Telegram.
- Все лиды попадают в одну общую систему.

## 3. Главное меню

- `Новые заявки (N)`
- `В работе (N)`
- `Отложенные (N)`
- `Просроченные (N)`
- `Сегодня (N)`
- `Завтра (N)`
- `Добавить лид`
- `Поиск`
- `Черный список (N)`
- `Напоминания: Вкл/Выкл`
- `Настройки`

## 4. Логика разделов

- `Новые заявки` — только новые лиды, по ним показывается дата и время поступления.
- `В работе` — активные лиды, по которым идет работа.
- `Отложенные` — лиды, у которых назначен следующий контакт на будущее.
- `Просроченные` — лиды, у которых следующий контакт уже просрочен.
- `Сегодня` — отдельные блоки `Касания` и `Записи`.
- `Завтра` — отдельные блоки `Касания` и `Записи`.
- Если касание стояло на сегодня, но время уже прошло, лид показывается и в `Сегодня`, и в `Просроченных`.
- `Записи` и `Касания` никогда не смешиваются в одну ленту.

## 5. Отображение лидов в списках

Каждый лид показывается в 2 строки:

```text
Иван | BMW X5 | Полировка
В работе | Горячий | сегодня 18:00
```

Для новой заявки:

```text
Сергей | Kia Rio | Химчистка
Новая заявка | — | поступил 18.04 14:32
```

Сверху в списках фильтр:

- `Все`
- `Горячие`
- `Теплые`
- `Холодные`

## 6. Карточка лида

Поля:

- Имя
- Телефон
- Авто
- Услуга
- Статус
- Температура
- Поступил
- Следующий контакт
- Источник
- Комментарий

Кнопки:

- `Связаться`
- `Результат`
- `Температура`
- `Комментарий`
- `История`
- `История клиента`
- `Создать запись`
- `Редактировать`
- `В ЧС`
- `Удалить`

## 7. Результаты по лиду

Кнопка `Результат`:

- `В работе`
- `Ждет решения`
- `Записан`
- `Не ответил`
- `Отложен`
- `Потерян`

Правила:

- `Новая заявка` показывает только время поступления.
- `В работе` может быть без следующего контакта.
- `Ждет решения`, `Не ответил`, `Отложен` требуют дату и время следующего контакта.
- `Записан` создает отдельную запись.
- `Потерян` требует причину и, при необходимости, комментарий.

## 8. Температура

- `Горячий`
- `Теплый`
- `Холодный`

Температура не является этапом, это отдельная характеристика лида. Она показывается в списке и в карточке.

## 9. Ручное добавление лида

Обязательно:

- Телефон
- Имя или пометка

Необязательно:

- Авто
- Услуга
- Комментарий
- Источник

Если номер уже существует, бот предупреждает и дает варианты:

- `Открыть существующий`
- `Создать новое обращение`
- `Отмена`

## 10. Повторы и дубли

Основа проверки дубля — нормализованный номер телефона.

Правила:

- Если номер в `Черном списке`, новый лид не создается.
- Если есть активный лид, новый лид не создается автоматически, открывается существующий.
- Если есть только старые закрытые обращения, можно создать новое обращение.
- Если лид был удален, он не участвует в проверке дублей.
- В карточке должна быть кнопка `История клиента`, которая показывает все обращения и записи по номеру.

## 11. История клиента

История клиента строится по номеру телефона и показывает:

- все обращения
- все записи
- статусы
- комментарии
- отметку о ЧС, если была

## 12. Записи

Запись — отдельная сущность, не смешивается с лидом.

Обязательные поля записи:

- Дата
- Время
- Услуга

Необязательные:

- Авто
- Комментарий
- Ожидаемая сумма
- Длительность

Статусы записи:

- `Назначена`
- `Подтверждена`
- `Перенесена`
- `В работе`
- `Выполнена`
- `Не приехал`
- `Отменена`

Статус записи можно менять в любом порядке.

Кнопки в карточке записи:

- `Изменить статус`
- `Перенести`
- `Комментарий`
- `Открыть лид`
- `История клиента`

## 13. Потерянные лиды

- `Потерян` — это финальный статус лида.
- В главное меню отдельный раздел `Потерянные` не выводится.
- Потерянные доступны через `Поиск` и `Историю клиента`.
- Потерянный лид можно вернуть в работу.

## 14. Удаление

- У лида есть кнопка `Удалить`.
- Удаление только через подтверждение.
- После удаления лид исчезает из рабочих разделов и поиска.
- Если номер снова придет с сайта или будет введен вручную, он считается новым лидом.

## 15. Черный список

В `Черный список` можно добавлять номер:

- из карточки лида
- вручную
- через поиск

У записи ЧС есть:

- Имя/пометка
- Телефон
- Причина
- Комментарий
- Дата добавления

Если номер из ЧС снова оставил заявку:

- обычный лид не создается
- приходит уведомление
- кнопки:
  - `Открыть ЧС`
  - `Убрать из ЧС и создать лид`

## 16. Поиск

Поиск ищет по:

- телефону
- имени
- услуге
- комментарию
- авто

Фильтры:

- `Все`
- `Активные`
- `Потерянные`
- `Черный список`

## 17. Уведомления

- `Новые заявки` — всегда мгновенный пуш, не отключается.
- Все остальное — одной ежедневной сводкой.
- По умолчанию сводка приходит в `10:00 МСК`.
- Время сводки можно менять в `Настройках`.
- `Напоминания: Вкл/Выкл` в главном меню включает и выключает только ежедневную сводку.
- Настройка сводки личная для каждого пользователя.

Формат сводки:

- Просроченные касания
- Новые заявки без обработки
- Сегодня: касания / записи
- Завтра: касания / записи
- Отложенные

## 18. Настройки

- Часовой пояс
- Время ежедневной сводки
- Ежедневная сводка: Вкл/Выкл
- Сотрудники

`Настройки` находятся в главном меню.

## 19. Доступ и сотрудники

Есть 2 типа пользователей:

- Владелец
- Сотрудник

Рабочий функционал у них одинаковый. Разница только одна:

- Владелец может добавлять и удалять сотрудников.

Сотрудник получает доступ так:

- открывает бота
- отправляет запрос
- владелец подтверждает или отклоняет

Удаление сотрудника:

- владелец отключает доступ сотруднику
- сотрудник сразу теряет доступ
- история его действий сохраняется

## 20. Интеграция с сайтом

Сайт отправляет в бот `POST` запрос с полями:

```json
{
  "name": "Иван",
  "phone": "+79991234567",
  "service": "Полировка",
  "comment": "Хочу завтра после 16:00",
  "source": "site"
}
```

Правила:

- Время поступления фиксирует сам бот.
- Поле страницы сайта не передается.
- Если услуга уже известна по форме, она приходит сразу.
- Бот сам проверяет ЧС и дубли.
