# Current Status

## Что уже реализовано

- отдельный Telegram mini CRM бот в своей папке, отдельно от сайта
- Node.js-проект без внешних runtime-зависимостей
- SQLite-хранилище на `node:sqlite`
- Telegram webhook endpoint
- API endpoint приема лидов с сайта
- роли: владелец и сотрудники
- запросы доступа и подтверждение новых пользователей
- главное меню и рабочие разделы
- лиды: создание, редактирование, статусы, температура, история, история клиента
- записи: создание, перенос, смена статуса, история
- поиск, черный список, повторные лиды, защита от дублей
- ежедневная сводка и мгновенные уведомления о новых заявках

## Что уже отполировано

- большая часть inline-навигации обновляет текущее сообщение вместо спама новыми
- `В меню` сбрасывает активную сессию, чтобы не было зависшего сценария поиска или ввода
- редактирование телефона защищено от:
  - номера из ЧС
  - номера другого активного лида
- повторный `/start` не шлет владельцу дублирующиеся карточки запроса доступа
- создание лида из ЧС снимает номер из ЧС только после успешного создания
- счетчики `Сегодня` / `Завтра` учитывают только активные записи
- ежедневная сводка ведет в обзор дня, а не сразу в касания
- текстовые шаги из inline-карточек по возможности редактируют текущий экран, а после ввода возвращают карточку в том же сообщении
- удаление лида больше не шлет отдельное подтверждение и отдельное меню: приходит одно сообщение с главным меню
- исправлены битые русские строки в редких ветках доступа и проверки телефона
- один Telegram-бот разделен на клиентский вход и CRM-доступ: `/start` отвечает клиенту, `/access` отправляет запрос сотрудника владельцу
- личные сообщения клиентов сохраняются отдельно от лидов и дают уведомление владельцу с кнопками `Ответить` / `Создать лид` / `Игнорировать`
- лид из личного сообщения создается вручную по кнопке; если в тексте есть телефон, он подставляется автоматически
- ответа клиенту по умолчанию нет: ответ уходит только вручную из карточки сообщения
- карточка сообщения показывает историю ручных ответов по этому сообщению
- ответить можно повторно из обработанного сообщения в разделе `Все сообщения`
- карточка сообщения показывает последние другие сообщения этого же Telegram-клиента
- если в сообщении нет телефона, создание лида показывает исходный текст и просит сначала ввести телефон
- в главном меню добавлена кнопка `Сообщения (N)` со счетчиком новых входящих
- раздел `Сообщения` показывает два входа: `Новые сообщения (N)` и `Все сообщения`
- клиентский `/start` показывает кнопку `Написать вопрос`; после нее клиент пишет обычным текстом

## Что покрыто тестами

- сброс сессии при возврате в меню
- защита от смены телефона на номер из ЧС
- защита от смены телефона на номер другого активного лида
- отсутствие дубля уведомлений владельцу при повторных запросах доступа
- безопасный сценарий создания лида из ЧС
- сохранение ЧС при аварии создания лида
- корректные счетчики активных записей на сегодня и завтра
- корректные кнопки в ежедневной сводке
- редактирование текущей карточки при callback-сценарии ввода комментария
- отсутствие отдельного лишнего сообщения после удаления лида
- клиентский `/start` не создает запрос доступа
- входящее Telegram-сообщение хранится отдельно и может быть превращено в лид
- входящее Telegram-сообщение не отправляет клиенту автоответ
- ручной ответ из карточки сообщения отправляется клиенту и снимает сообщение с новых
- история ручных ответов отображается в карточке сообщения
- повторный ответ из обработанного сообщения
- история других сообщений того же Telegram-клиента
- создание лида из сообщения без телефона с понятным prompt-экраном
- счетчик новых Telegram-сообщений в главном меню
- экран сообщений с новыми и всеми сообщениями
- клиентская кнопка `Написать вопрос` после `/start`
