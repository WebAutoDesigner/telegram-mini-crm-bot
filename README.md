# Telegram Mini CRM Bot

Отдельный проект Telegram mini CRM-бота для бизнеса: принимает лиды с сайта и вручную, ведет карточки лидов и записей, поддерживает черный список, доступ сотрудников и ежедневные сводки.

Основные документы:

- [docs/final-spec.md](docs/final-spec.md) — финальная продуктовая логика бота
- [docs/current-status.md](docs/current-status.md) — текущее состояние реализации, проверки и следующие шаги

## Что уже собрано

- отдельный Node.js-проект без внешних зависимостей
- SQLite-хранилище на `node:sqlite`
- webhook-сервер для Telegram и endpoint приема лидов с сайта
- базовая access-логика: владелец, сотрудники, запросы доступа
- главное меню и основные разделы
- карточки лидов и записей
- ручное добавление лида
- прием лидов с сайта и мгновенные пуши
- ежедневная сводка по расписанию пользователя
- поиск, черный список, история клиента
- тестовый контур: `npm test` и `npm run smoke`

## Текущее состояние

Уже реализованы и проверены:

- сброс зависающих сессий при возврате в меню
- защита редактирования телефона от дублей и номеров из ЧС
- корректная логика повторных запросов доступа
- безопасный сценарий создания лида из номера из ЧС
- корректные счетчики активных записей на сегодня и завтра
- inline-навигация для большинства рабочих экранов без лишнего спама сообщениями

## Быстрый старт

1. Скопируйте `.env.example` в `.env`
2. Заполните:
   - `BOT_TOKEN`
   - `BOT_PUBLIC_URL`
   - `BOT_WEBHOOK_SECRET`
   - `SITE_API_KEY`
   - `OWNER_TELEGRAM_ID`
3. Запустите:

```bash
npm start
```

Для разработки:

```bash
npm run dev
```

Проверки:

```bash
npm test
npm run smoke
```

## HTTP endpoints

- `GET /health`
- `POST /telegram/webhook/{BOT_WEBHOOK_SECRET}`
- `POST /api/leads`

Для `/api/leads` нужен заголовок:

```text
X-Api-Key: <SITE_API_KEY>
```

Тело запроса:

```json
{
  "name": "Иван",
  "phone": "+79991234567",
  "service": "Полировка",
  "comment": "Хочу завтра после 16:00",
  "source": "site"
}
```
